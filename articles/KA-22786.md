# 作成者で DAM アセットコンソールを読み込めません


オーサー環境で Assets AEMコンソールにアクセスする際に空白の画面が表示される場合は、ワークフローインスタンスが多すぎてエラーが発生する可能性があります `[` 1`]` . これらのインスタンスをクリーニングすることで解決できます。

## 説明 {#description}


### 環境

Adobe Experience Manager(AEM) オンプレミス 6.5.15

### 説明

「 」を使用して Assets AEMコンソールにアクセスしようとしたときに、`/assets.html/content/dam`」画面は空白です。


## 解決策 {#resolution}


ワークフローインスタンスをクリーニングした後で、問題が修正されました。

### 原因

エラーログを確認中に、エラーが発生しました `[` 1`]`  が見つかりました。

Assets UI と Sites UI は、リストする項目のワークフローステータスを参照する必要があります。 したがって、ワークフローインスタンスが多すぎると、ページの読み込みに失敗し、エラーが発生します `[` 1`]`  がログに表示されます。
