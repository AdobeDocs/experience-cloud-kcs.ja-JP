# Mixin の競合（およびクラシック UI の使用）が原因でロールアウトが失敗する


Mixin とコンテンツ更新アクション設定の競合（クラシック UI を使用）が原因でロールアウトが失敗する問題を修正する方法を説明します。

## 説明 {#description}


一部の競合が原因でページのロールアウトが失敗します *jcr:mixinTypes* ページのサムネールノードで上書きできます。

競合は、ページがサムネールで設定された場合に発生します <b>クラシック UI の使用</b> その後、最初にロールアウトします（最初のロールアウトは正常に行われます）。

この時点で、ページのサムネール構造の次の範囲で mixin が競合していることが原因で、マスターページのロールアウトの実行に失敗します。

- dam：サムネール
- cq:LiveRelationship

<b>再現手順：</b>

1. vanilla AEM 6.5.13.0 を起動します。
2. に移動 <b>/siteadmin</b> （を使用 <b>クラシック UI</b>）
3. のページプロパティを開く */content/we-retail/language-master/en/women*
4. イメージ内 *タブ* 任意の画像をサムネールとして設定
5. ページをロールアウトします
6. 次に進む <b>Sites UI</b>
7. のページプロパティを開きます。 */content/we-retail/us/en/women* 再び
8. が含まれる *サムネール* タブで、をクリックします。 *プレビューを生成* 変更を保存します
9. 次に、/siteadmin 内のページを再度ロールアウトします



## 解決策 {#resolution}


この問題は、AEMがライブコピーページの jcr:content （/content/we-retail/us/en/women/jcr:content/image/file/jcr:content）の下に「dam:thumbnails」ノードを作成できず、ロールアウト中に「dam:Thumbnails」がライブコピーページのノード「/content/we-retail/us/en/women/jcr:content」に適用されず、「dam:Thumbnails」フォルダーが「dam:Thumbnails」 nodeType のの下にのみ作成されます。

MSM のデフォルトの動作では、ロールアウト時にソースのノードに追加された新しい Mixin は無視されます `[` 0`]`  したがって、「/content/we-retail/us/en/women/jcr:content/image/file/jcr:content」には追加されません。

この問題を修正するには、コンテンツ更新アクションの設定を使用して、この動作を変更できます。
以下のスクリーンショットに示すように、デフォルトでは、すべての Mixin タイプが無視されます。

![](assets/6e22f175-e313-ed11-b83d-002248086a27.png)

次を使用するように更新する必要があります。


```
(?!.*dam:Thumbnails).*
```


次のスクリーンショットに示します。

![](assets/4d5f7db7-e313-ed11-b83d-002248086a27.png)

これで問題は解決するはずです。

`[` 0`]` https://experienceleague.adobe.com/docs/experience-manager-64/administering/introduction/msm-sync.html?lang=en#excluding-properties-and-node-types-from-synchronization
