## 解決策 {#resolution}


この問題は、データストアを使用する以前のインスタンス（破棄され、データストアを使用するために新しいインスタンスが作成された）がある場合に発生する可能性があります。

この問題は、データストアに登録されているリポジトリ ID を削除すると解決できます。 BLOB ストア GC は正常に完了します。

ステップバイステップガイド：

1）データストアのパスに移動  `<` aem-install-folder`>` /crx-quickstart/repository/datastore とリポジトリ id の検索 <b>e0fb2a15-d5d5-4013-9607-0fb89cf87c94</b> （error.log に記載されているものと同じである必要があります。 <b>エラー</b>\* `[` sling-oak-observation-19`]`  org.apache.jackrabbit.oak.plugins.blob.MarkSweepGarbageCollector すべてのリポジトリに使用可能な参照があるわけではありません。 `[` <b>e0fb2a15-d5d5-4013-9607-0fb89cf87c94</b>`]` ）を選択すると、この名前の参照ファイルが作成されます。その参照ファイルを削除するだけです（複数のファイルがある場合は、これらの手順を繰り返します）。

2）もう一度 GC を実行し、問題が解決したかどうかを確認します。
