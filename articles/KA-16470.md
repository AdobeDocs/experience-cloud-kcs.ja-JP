# Audience Manager先 [!DNL Campaign]

## 説明


このテクニカル実装ガイドでは、Audience Managerと [!DNL Campaign].



このテクニカル統合ガイドは、Adobeコンサルタント、Adobeパートナー、その他の経験豊富な実装担当者を対象としています。 この統合を設定するには、このガイドで説明する各ソリューションの実装プロセスを完全に理解している必要があります。

Adobeのお客様が [Adobe Professional Services](http://www.adobe.com/experience-cloud/service-support/professional-services/contact-us.html) または [Adobeパートナー](http://www.adobe.com/experience-cloud/service-support/partners.html) この統合の詳細を確認するには、を参照してください。




## 解像度


<b>統合の概要</b>

この統合では、AAM内でMarketing Cloudの共有オーディエンスと宣言済み ID 機能を使用します。

<b>ソリューションの計画と統合アーキテクチャ</b>

A.前提条件（ライセンス、接続、初期化）

1. Marketing Cloud用にプロビジョニングされた顧客組織
2. Adobe Analyticsライセンス
3. Adobe Campaign V6 を 8687 以上にアップグレード（8705 以降を推奨）
4. Adobe Audience Managerライセンス
5. Adobe DTM
6. Adobe Marketing Cloud共有オーディエンス
7. Adobe Campaignにプッシュする顧客 CRM データ
8. AdobeCampaignv6 は、Marketing-Cloud と連携する必要があります
9. 設定 [!DNL Campaign] Marketing Cloud
10. ユーザーが [!DNL Campaign] Adobe ID経由の V6

<b>インストール、プロビジョニング、および設定</b>

A.セットアップ/プロビジョニング

1. その他のソリューションコアサービスのプロビジョニングは、Adobe担当者を通じておこなう必要があります。
2. [!DNL Campaign] プロビジョニングは、次の方法でおこなう必要があります。 [!DNL Campaign] TechOps.
3. AAM UI とAAM FTP のデータソース、宛先は、Adobe担当者を通じて作成またはプロビジョニングする必要があります。


B.ソリューションの実装

1. 顧客 CRM

すべての CRM データを保存するシステムが必要です。 これは、電子メールアドレスのリストと、この顧客の一意のハッシュ化された ID を組み合わせる必要があります。 例： *[abc@abcmail.com](mailto:abc@abcmail.com), 1231231j2g3123<b>[xyz@xyzmail.com](mailto:xyz@xyzmail.com), 234234jh23k4jh324* 最初の列は電子メールアドレス、2 番目の列はハッシュ化された CRM_ID です

2. CRM データベースをと同期 [!DNL Campaign] 顧客は、CRM データをにプッシュするプロセスを設定する必要がある [!DNL Campaign]. 電子メール、 CRM_ID は、同期される最小フィールドです。

3. ページで CRM_ID を使用可能にします。この場合は、顧客がログインしている必要があります。 ログイン時に、CMS は顧客 CRM に連絡して、パスワードが正しいことを確認する必要があります。 このレベルでは、一意の CRM_ID も取り込んでページ、できればデータレイヤーに送信する必要があります。

4. DTM は、「Marketing CloudID サービス」ツールにある CRM_ID 設定を読み取ります。 hashed_email データ要素は、dataLayer で使用可能な CRM_ID を読み取る DTM で設定されます。

5. Adobe CampaignでのAAM FTP の設定

AAM用に作成された SFTP は、次のようにAdobe Campaignで設定する必要があります。 [!DNL Campaign] AAMからセグメントをプッシュおよびプルできます。 SFTP 資格情報は、AdobeカスタマーケアまたはAdobe担当者から提供されます。 プロビジョニング中に、ftp-in と ftp-out の両方の場所がプロビジョニングされ、 [!DNL Campaign] オーディエンスをAAMに送信し、AAMからにセグメントをオンボーディングする [!DNL Campaign] それぞれ

6. AAMでのデータソースと宛先の作成

統合用のデータソースと宛先は、AAM UI で作成する必要があります（まだ作成していない場合）。 この機能が作成されている場合は、データソースと宛先 ID を使用して、Adobe Campaign UI で設定する必要があります。 2 つのデータソースを作成する必要があります。1 つはMarketing CloudID に基づき、もう 1 つは宣言済み ID に基づきます。 宛先は、各データソースに対応するAAM Admin UI で作成されます。 これには、AAMコンサルタントまたはAAM Clientcare の関与が必要になります。

![4](https://helpx.adobe.com/content/dam/help/en/marketing-cloud/how-to/aam-campaign/_jcr_content/main-pars/procedure/proc_par/step_1253393284/step_par/image/4.jpg "4")

7. Adobe Campaign内でデータソースを設定するデータソースはAdobe Campaignで設定されます。 2 つのデータソースがAdobe Campaign UI で作成されます。1 つは宣言された ID またはハッシュ化された顧客 ID に基づき、もう 1 つはMarketing CloudID または要求者 UUID に基づきます。 各 ID に対応する宛先 ID は、各フィールドに追加されます。 例えば、AAMの宛先Adobe Campaign — 宣言済み ID の宛先 ID が、ハッシュ化された顧客 ID として紐付けキーと共に、データソース設定に追加されます。

Adobe IDを使用してログインしていない場合は、 [!DNL Campaign]. Adobe IDを使用してログインした場合は、Enterprise Console UI で Audience Library 製品にMarketing Cloudされていることを確認してください。

- セグメントをAAMと共有しましたが、AAM UI にまだ表示されません。 理由は？
