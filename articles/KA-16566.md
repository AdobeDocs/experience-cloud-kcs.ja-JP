# Oak Lucene インデックスを最適化してインデックス作成時間を短縮する方法

## 説明


AEM Oak Lucene インデックスを最適化して、インデックス再作成時間を短縮します。
<br><br>環境<br><br>
AEM 6.0 + Oak 1.0.16 以降のAEM 6.1 およびAEM 6.2


## 解像度


「includedPaths」、「excludedPaths」、「name」、「indexPath」などのプロパティを使用して oak: QueryIndexDefinition を微調整することで、Oak Lucene インデックスを最適化できます。

- &quot;includedPaths&quot;プロパティと&quot;excludedPaths&quot;プロパティは、インデックス再作成時にインデックスが通るパスと、インデックス更新のためにリッスンするパスを制限します。
- 「name」および「indexPath」プロパティを使用すると、インデックスのパスまたは名前でログメッセージにタグ付けするので、インデックスログを理解しやすくなります。  例：



| `30.01.2017 14:07:12.934 *DEBUG* pool-8-thread-2 org.apache.jackrabbit.oak.plugins.index.lucene.LuceneIndexEditor damassetlucene(` `/oak` `:index` `/damAssetLucene` `) = Indexed 9000 nodes...` |
| --- |


<b>インデックスを最適化する手順</b>:

1. http://に移動します。*aem-host:port*/crx/de/index.jspを開き、管理者としてログインします。
2. /oak:index を参照します。
3. 1 の下で強調表示されているプロパティを、対応するインデックスに追加します。  この <b>includedPaths</b> および <b>excludedPaths</b> プロパティは、複数値の文字列 ( String ) である必要があります
4. 「[!UICONTROL すべて保存]CRXDe の「 」
